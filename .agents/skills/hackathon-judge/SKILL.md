Read `docs/HACKATHON_REQUIREMENTS.md`, `docs/CODEX_MASTER_PLAN.md`, and the current implementation. Review as a skeptical hackathon judge.

For each criterion:
1. Identify what a judge can directly see and verify.
2. Identify implementation evidence, not marketing language.
3. Flag theatrical AI, hidden fake data, unsupported claims, incomplete flows, or weak differentiation.
4. Recommend the smallest high-leverage fix.
5. Re-score only after verifying code/tests/demo evidence.

Never award credit for a feature that is only described in documentation or seeded traces.
