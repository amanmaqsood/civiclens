# CivicLens Demo Script

Use this as a judge-facing flow. Keep every statement truthful: CivicLens is a prototype, not a government portal.

## Opening

"CivicLens helps citizens document local civic issues, groups possible duplicates, asks the community for verification, and gives a human operator a review workspace with AI-drafted recommendations. It does not file complaints automatically or claim government acknowledgement."

## Flow

1. Start on the citizen view and point out the prototype/non-government boundary.
2. Submit a report with a photo, description, and location.
3. Show Gemini-assisted draft analysis and clarify that the citizen can edit before saving.
4. Show duplicate comparison. Merge only if the human chooses it.
5. Open the saved case and show community support/verification.
6. Run the persisted server agent. Point out that the UI renders stored `agentRuns`/`agentSteps`, not client-made traces.
7. Open the operator desk.
8. Review evidence, priority, recommendation, and draft routing/action packet.
9. Record human approval for routing/action packet.
10. Draft escalation/RTI text and point out it is not submitted externally.
11. Upload closure evidence.
12. Show closure assessment recommendation and make the final resolve/reopen decision as a human operator.
13. Open the dashboard and explain that metrics are calculated from loaded persisted records, separated between real and synthetic demo data.

## Closing

"The value is not replacing civic authorities. The value is better citizen evidence, duplicate reduction, transparent AI assistance, and a human-governed review trail that can be handed to the right authority outside the app."
